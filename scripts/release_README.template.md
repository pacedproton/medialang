## Quick start

### Read the data without any database

The CSV files in `02_csv_tables/` open directly in R, Python, DuckDB, or
spreadsheet tools. Example with DuckDB:

```sql
SELECT * FROM read_csv_auto('02_csv_tables/mo_constant.csv') LIMIT 10;
```

### Restore the original PostgreSQL database

```bash
createdb anmi
psql anmi < 01_source_database/pg_dump_graphv3_full.sql
```

The dump contains the `graphv3` schema with all 17 tables and data, but no
roles, ownership, or privileges (it restores cleanly under any user).

### Re-derive the MDSL representation from the source database

See `04_compiler/README.md` for the compiler version and exact command.

## What's in the source database

The `graphv3` schema contains 17 tables in three groups:

| Table | Group | Description |
|---|---|---|
| `mo_constant` | core | One row per media outlet (time-invariant attributes) |
| `mo_year` | core | Annual market metrics per outlet (circulation, reach, etc.) |
| `nd_company`, `nd_person`, `nd_ownership` | actors | Companies, natural persons, ownership relations |
| `nd_crawl_log` | actors | Provenance log for ownership data |
| `sources_names` | provenance | Catalog of data sources cited by metric records |
| `11_succession`, `12_amalgamation`, `13_new_distribution_area`, `14_new_sector`, `19_interruption`, `21_split_off`, `22_offshoot`, `23_merger` | diachronic relations | Temporal (predecessor/successor-style) relationships between outlets |
| `31_main_media_outlet`, `33_umbrella`, `34_collaboration` | synchronous relations | Contemporary (overlapping-period) relationships between outlets |

Detailed variable definitions for each table are in
`05_documentation/codebook.pdf`.

### Note on table names beginning with digits

Table names like `11_succession` are accepted by PostgreSQL but require
quoting in some SQL dialects and may be auto-renamed by R (`X11_succession`)
or rejected by older tools. The names are preserved verbatim from the source
database for provenance reasons.

### Note on absent primary key constraints in market-data tables

`MedienangebotMarktdaten`, `MedienunternehmenMarktdaten`, and
`MediensektorNutzungsdaten` (in the MDSL representation under
`03_mdsl_representation/MediaLanguage/anmi_market_data_schemas.mdsl`)
intentionally omit explicit `PRIMARY KEY` constraints. This is a deliberate
modeling choice: rows are identified by the composite of entity, year, and
source, and the schema permits multiple overlapping records from different
sources for the same entity-year. Adding a synthetic single-column primary
key would obscure that the source data is multi-sourced and potentially
discordant.

## Provenance

This release was generated as follows:

1. Source database: PostgreSQL {{PG_VERSION}}, schema `graphv3`,
   snapshotted on {{DATE}}.
2. SQL dumps: `pg_dump --schema=graphv3 --no-owner --no-privileges`.
3. CSV exports: `\COPY (SELECT * FROM graphv3.<table>) TO ... WITH CSV HEADER`,
   one file per table.
4. MDSL representation: schema files curated by the project team
   (`03_mdsl_representation/MediaLanguage/`) plus a bulk export
   (`anmi_full_export.mdsl`) generated by the ANMI-ML compiler from the
   PostgreSQL source.
5. Validation: every MDSL file in this release validates with zero errors
   under ANMI-ML compiler version `{{COMPILER_VERSION}}` (see
   `05_documentation/validation_run.log`).

## Ethics and personal data

This dataset describes media institutions and their corporate ownership
structures. Where natural persons appear (e.g. board members, ownership
records in `nd_person` and `nd_ownership`), the information is derived from
public registries and does not include contact details, dates of birth, or
other directly identifying private information. Reusers in EU/EEA
jurisdictions should ensure that any onward processing complies with GDPR
Art. 6 lawful-basis requirements (typically Art. 6(1)(e) public interest
or Art. 6(1)(f) legitimate interests for journalistic/academic research,
plus Art. 89 safeguards for archiving in the public interest).

## Reproducibility and verification

To verify file integrity:

```bash
sha256sum -c checksums.sha256
```

To re-run the validation contract (requires the ANMI-ML compiler):

```bash
mdsl validate 03_mdsl_representation/MediaLanguage/anmi_main.mdsl
```

Expected output: `Status: PASSED, Errors: 0`.
